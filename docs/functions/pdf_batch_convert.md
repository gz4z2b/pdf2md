# 批量 PDF 转 Markdown

## 功能用途

本功能用于把 `origin` 目录以及子目录里的 PDF 文件批量转换成 Markdown 文档。

适合下面这种场景：

1. 用户有很多 PDF 文档。
2. 用户想把这些 PDF 变成更容易阅读、复制和二次编辑的 Markdown。
3. 用户不想一个文件一个文件手动转换。

## 使用入口

开发完成后，用户执行：

```bash
bash scripts/convert.sh
```

脚本会自动扫描 `origin` 目录，不需要用户记复杂命令。

## 输入目录

默认输入目录：

```text
origin/
```

规则：

1. 只处理 `.pdf` 文件。
2. 会递归处理子目录。
3. 文件后缀大小写不敏感，`.pdf` 和 `.PDF` 都应识别。
4. 非 PDF 文件会被跳过。

示例：

```text
origin/
├── a.pdf
├── b.PDF
└── sub/
    └── c.pdf
```

## 输出目录

默认输出目录：

```text
output/
```

输出时会保留原来的子目录结构：

```text
output/
├── a.md
├── b.md
└── sub/
    └── c.md
```

## 转换规则

| PDF 内容 | Markdown 输出规则 |
| --- | --- |
| 普通文字 | 输出为 Markdown 段落 |
| 页面分隔 | 使用 `## 第 N 页` |
| 明显标题 | 尽量转换为 Markdown 标题 |
| 列表文字 | 尽量保留换行和缩进 |
| 表格内容 | v1.0.0 尽量保留为可读文本，后续再增强为标准 Markdown 表格 |
| 图片 | 输出图片占位符 |
| 空白页 | 输出空白页提示 |

图片占位符示例：

```markdown
[图片占位符：第 1 页第 2 张图片]
```

空白页提示示例：

```markdown
[本页没有可识别文字]
```

## 参数说明

v1.0.0 默认不要求用户填写参数。代码实现时可以预留下面这些参数，方便后续扩展。

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--input-dir` | `origin` | PDF 输入目录 |
| `--output-dir` | `output` | Markdown 输出目录 |
| `--overwrite` | `false` | 是否覆盖已经存在的 Markdown 文件 |
| `--log-level` | `INFO` | 日志详细程度 |

## 返回结果

一键脚本执行结束后，终端应展示汇总信息：

```text
完成：成功 3 个，失败 0 个，输出目录：output
```

如果有失败文件，应展示失败列表：

```text
完成：成功 2 个，失败 1 个
失败文件：
- origin/bad.pdf：文件无法读取
```

## 异常处理

| 情况 | 处理方式 |
| --- | --- |
| `origin` 不存在 | 自动创建目录，并提示用户放入 PDF 后重新执行 |
| 没有 PDF 文件 | 输出提示，不报错 |
| PDF 损坏 | 记录失败原因，继续转换其他文件 |
| 输出目录不存在 | 自动创建 |
| 输出文件已存在 | 默认不覆盖，除非用户启用覆盖参数 |

## v1.0.0 不做的内容

下面内容不属于 v1.0.0 范围：

1. 不导出 PDF 中的真实图片文件。
2. 不保证复杂表格 100% 转成标准 Markdown 表格。
3. 不处理扫描版 PDF 的 OCR 识别。
4. 不提供网页界面。
5. 不提供批量上传下载服务。

这些能力可以放到后续版本继续开发。
