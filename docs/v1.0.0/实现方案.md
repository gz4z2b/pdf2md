# v1.0.0 实现方案

## 原始需求理解

v1.0.0 要解决的问题是：用户把 PDF 文件放进 `origin` 目录后，只执行一个脚本，就能批量得到 Markdown 文件。如果 `origin` 里本来就有 Markdown 文件，也应该一起复制到 `output` 中。

用户最关心的是：

1. 不需要复杂操作。
2. 能处理 `origin` 子目录里的 PDF 和 Markdown。
3. Markdown 尽量保持 PDF 原来的阅读顺序和排版。
4. PDF 中能提取到的图片需要导出到 `output/sources`，并在 Markdown 中引用。
5. 脚本能自动检查环境和安装依赖。

## 产品方案

提供一个命令行批量转换工具，用户只需要使用：

```bash
bash scripts/convert.sh
```

默认约定：

| 项目 | 约定 |
| --- | --- |
| 源文件输入目录 | `origin` |
| Markdown 输出目录 | `output` |
| 文件匹配 | 递归查找所有 `.pdf` 和 `.md` 文件 |
| 输出文件名 | PDF 后缀改为 `.md`；文件名末尾如果有 `.note`，输出时去掉；Markdown 源文件也使用同样规则 |
| 子目录 | 保留原来的相对路径 |
| 已存在输出文件 | 默认直接替换为最新转换结果 |
| 图片处理 | 导出到 `output/sources/源文件相对路径去后缀/`；资源目录名末尾如果有 `.note`，输出时去掉 |
| Markdown 源文件 | 直接复制，不改内容 |

## 技术栈

| 技术 | 选择 |
| --- | --- |
| 开发语言 | Python 3.11+ |
| PDF 解析库 | PyMuPDF |
| 命令行工具 | argparse |
| 日志工具 | logging |
| 自动化测试 | pytest |
| 测试覆盖率 | pytest-cov |
| 测试 PDF 生成 | reportlab |

选择 PyMuPDF 的原因：

1. 能读取 PDF 文本块和页面结构。
2. 能识别页面中是否存在图片。
3. 安装和使用都比较直接，适合 v1.0.0。

## 架构设计

```text
scripts/convert.sh
        |
        v
src/pdf2md/cli.py
        |
        v
src/pdf2md/batch.py
        |
        +--> src/pdf2md/pdf_reader.py
        |
        +--> src/pdf2md/markdown_writer.py
```

模块职责：

| 模块 | 职责 |
| --- | --- |
| `scripts/convert.sh` | 一键执行入口，负责环境检查、安装依赖、启动程序 |
| `cli.py` | 读取命令行参数，调用批量转换流程 |
| `config.py` | 保存默认输入目录、输出目录和运行配置 |
| `batch.py` | 扫描 PDF 和 Markdown 文件，逐个处理，汇总成功失败结果 |
| `pdf_reader.py` | 读取 PDF 页面、文字块、表格和图片数据 |
| `markdown_writer.py` | 把 PDF 解析结果写成 Markdown，并写入图片资源文件 |
| `logger.py` | 控制日志格式 |
| `errors.py` | 定义项目异常，方便给用户友好提示 |

## 转换流程

1. 用户把 PDF 或 Markdown 放入 `origin`。
2. 用户执行 `bash scripts/convert.sh`。
3. 脚本检查 Python 版本。
4. 脚本创建 `.venv` 虚拟环境。
5. 脚本安装 `requirements.txt` 中的依赖。
6. Python 程序递归扫描 `origin`。
7. 对每个 PDF：
   - 打开 PDF。
   - 按页读取文本。
   - 识别并提取页面图片。
   - 生成 Markdown 页面内容。
   - 把图片保存到 `output/sources`。
   - 写入 `output` 对应路径。
8. 对每个 Markdown：
   - 不解析内容。
   - 直接复制到 `output` 对应路径。
9. 程序输出成功和失败汇总。

## Markdown 格式设计

每个 PDF 输出一个 Markdown 文件。

示例：

```markdown
# demo

## 文档标题

这里是第一页文字。

1. 第一项
2. 第二项

| 字段 | 内容 |
| --- | --- |
| 示例 | 表格内容 |

![图片：第 1 张图片](sources/demo/page-001-image-001.png)
```

转换时不额外写入 `## 第 N 页` 这类页码标题。

## 一键脚本设计

`scripts/convert.sh` 应完成：

1. 切换到项目根目录。
2. 检查 Python 是否存在。
3. 检查 Python 版本是否满足 3.11+。
4. 如果 `.venv` 不存在，则创建虚拟环境。
5. 安装或更新依赖。
6. 启动转换命令。
7. 把执行结果打印给用户。

## 测试方案

开发时需要先写测试，再写实现代码。

重点测试：

1. 单个 PDF 转换。
2. 多个 PDF 批量转换。
3. 子目录 PDF 转换。
4. PDF 图片导出和 Markdown 图片链接生成，无法导出时保留占位符。
5. 空目录处理。
6. 损坏 PDF 处理。
7. 输出目录自动创建。
8. Markdown 源文件直接复制。
9. 一键脚本可执行。
10. PDF 图片导出到 `output/sources`，Markdown 图片链接可打开。
11. 源文件名末尾带 `.note` 时，输出文件和图片目录去掉 `.note`。

详细规范见 [../test.md](../test.md)。
